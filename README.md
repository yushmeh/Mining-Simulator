# Mining Simulator (Майнинг)

**Mining Simulator** — это интерактивное игровое веб-приложение, созданное для развлечения. Игрок может погрузиться в игровой процесс добычи разнообразных полезных ископаемых, улучшать свои инструменты, расширять вместимость рюкзака и зарабатывать игровую валюту.

---

## Основные функции

* **Динамическая система добычи:** Реалистичный кликер с визуальной отдачей (эффект покачивания руды при ударе).
* **Разнообразие ресурсов:** Система видов, редкости, прочности (HP) и стоимости камней (Камень, Уголь, Железо, Сапфир, Рубин, Аметист, Алмаз).
* **Кастомизация интерфейса:** Автоматическое изменение цветовой гаммы интерфейса, неонового свечения и полоски здоровья (HP) в зависимости от типа добываемой в данный момент руды.
* **Инвентарь (Рюкзак):** Наглядное отображение текущего количества собранных ресурсов, с ограничением по максимальной вместимости слотов.
* **Внутриигровой Магазин:** Покупка улучшений для кирок и рюкзаков за заработанные монеты с мгновенным обновлением параметров.
* **Интегрированная система уведомлений:** Всплывающие прямо внутри игрового интерфейса уведомления о добыче руды, успешных покупках или переполнении рюкзака.

---

## Архитектура проекта

Проект построен на архитектуре разделения логики бэкенда (Python/Flask) и интерактивного отображения (HTML/CSS/JS):

```text
├── app/
│   ├── __init__.py       # Инициализация и создание Flask-приложения
│   ├── routes.py         # Обработка маршрутов страниц и игровых API-эндпоинтов
│   ├── models.py         # Модели данных (Игрок, Камни) и сериализация данных
│   └── game_logic.py     # Ключевая бизнес-логика игрового процесса
├── static/
│   ├── css/
│   │   └── style.css     # Стили оформления, темы руд и анимация покачивания
│   ├── js/
│   │   └── main.js       # Асинхронное взаимодействие (Fetch API) и обновление UI
│   └── images/           # Папка для графических текстур и картинок руды
├── templates/
│   └── index.html        # Главный шаблон игрового интерфейса (Jinja2)
├── config.py             # Конфигурационные параметры Flask (включая SECRET_KEY)
├── requirements.txt      # Список зависимостей проекта (Flask и др.)
└── run.py                # Точка входа для запуска веб-сервера
```

## Объектно-ориентированная структура (Классы)

* **Player** — Модель игрока. Хранит баланс монет, структуру инвентаря, текущую экипированную кирку и активный камень для добычи. Поддерживает сохранение состояния сессии.
* **Stone** — Объект добываемой руды. Отвечает за генерацию параметров прочности (HP), базовой стоимости, названия и уникального системного ID.
* **Pickaxe** — Модификатор инструмента. Определяет силу удара (наносимый урон по HP камня) и стоимость улучшения.
* **Inventory** — Контроллер рюкзака. Следит за лимитом доступных слотов, подсчитывает общее количество предметов и отвечает за добавление ресурсов.
* **Upgrade** — Обобщенный класс улучшений. Реализует логику прокачки уровней инструментов (Кирка) и расширения вместимости (Рюкзак).
* **Game** — Управляющий класс. Связывает воедино всю механику: расчёт шансов выпадения руды, транзакции в магазине и калькуляцию прибыли при продаже.

## Технологический стек

**Backend:** Python 3.14, Flask (сессии для хранения прогресса игрока)
**Frontend:** HTML, CSS, JavaScript

## Установка и запуск
Установите необходимые зависимости:

```Bash
pip install -r requirements.txt
```

Запустите Flask-сервер:

```Bash
python run.py
```

Откройте в браузере локальный адрес: http://127.0.0.1:5000/
